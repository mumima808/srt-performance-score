## アーキテクチャ

言語が先にある。映像はその上に載る。

```
[ Audio / Language / Story ]
          │
          ▼
        Vrew
   ことばの一塊へ分解
   タイムスタンプを付与
          │
          ▼
  ┌─────────────────┐
  │  SRT = Performance Score  │
  │  sequence · timecode · text  │
  └─────────────────┘
          │
    ┌─────┼─────┬──────────┐
    ▼     ▼     ▼          ▼
DaVinci  TTS  LLM +     自動
Resolve       AIキャラ   動画生成
  映像    声   演技・     ワンソース
  編集   生成  パフォーマンス マルチユース
```

SRTはHUBだ。言葉とタイムコードだけを書けば、あとはツールが演奏する。

## 系譜 — これはずっとあった構造だ

SRTパフォーマンス・スコアは発明ではない。何世紀にもわたって、あらゆるジャンルに存在していた構造の**発見**だ。

| 形式 | 時間の設計 |
|------|------------|
| 紙芝居 | 絵と語りの間 |
| 4コマ漫画 | 起承転結のリズム |
| 落語 | 「えぇ…」の沈黙 |
| ヒップホップ | コール＆レスポンス |
| テクノ | ビルド＆ドロップ |
| VJライティング | 照明キュー |
| YouTube viral | 最初の3秒の設計 |
| MV | リズムカット |
| マッシュアップ | つなぎのタイミング |
| **SRT** | **← これら全ての最小単位** |

すべては、**時間をまたいだ緊張と解放の設計**だ。SRTはその設計を、機械が読める最小の形にエンコードしたものだ。

## 誰のためか

このプロジェクトは、すでに満足している人のためではない。

- **MVが好きだが、作り方を知らない人**  
  リズムを感じている。カットが頭の中で見えている。ツールを知らないだけだ。

- **マッシュアップが頭の中で鳴っているが、楽譜が読めない人**  
  誰も聴いたことのない組み合わせが聴こえている。スコアがそれを可視化する。

- **物語があるが、映像に変換できない人**  
  物語は存在している。時間になるためのプロトコルが必要なだけだ。

- **ローカルでしか作業しない普通のプログラマー**  
  GitHubが怖い（このプロジェクトの作者もそうだった）。でも構造的に考えている——それが本当の仕事だ。

> コンセプトを見る人——インターフェースではなく——が、  
> 他の全員が見逃したものを見つける。

## ロードマップ

### Phase 1 · 今 · 命名
```
名付ける。書く。旗を立てる。
─────────────────────────────
□ noteとXにコンセプトを公開する
□ GitHubリポジトリをマニフェストとして育てる
□ 語彙を確立する：SRT = プロトコル、スコア、テンプレート
```

### Phase 2 · 近い将来 · 実験空間
```
実験の場を作る。
─────────────────────────────
□ GitHubリポジトリをリビング設計書として運用
□ 集まる場所を作る（Discord等）
□ 最初の実験：SRT → LLM → AIキャラクター出力
```

### Phase 3 · 成長期 · 誰でも使えるツールへ
```
他の人が手に持てるものにする。
─────────────────────────────
□ 紙芝居・MV・マッシュアップ・短編映画のテンプレート
□ LLM連携のSRTジェネレーターパターン
□ 「まだできない人」のためのドキュメント
```

### Phase 4 · 未知 · 広がった先
```
設計できない。
─────────────────────────────
創発は、定義上、計画されない。
条件を作る——そして、見守る。
```
